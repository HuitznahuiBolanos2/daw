Definiciones:


HTTP:

HTTP es un protocolo de intercambio de informacion que se de fine como "Hyper Text Transfer Protocol" con la velocidad necesaria para compartir informacion. El protocolo ha estado en uso desde 1990. HTTP permite especificar el proposito de varias peticiones al sistema.[1]



URL:

La URL o "Uniform Resource Location" es un sistema de identificacion de recursos que permite encontrar la localidad del recurso deseado. Existe un documento que describe las reglas que se deben seguir al momento de crear una "URL" que describa la localidad de un recurso y otras de sus caracteristicas.[2]



MIME

MIME es un acronimo que se refiere a "Multipurpose Internet Mail Extensions" y es un protocolo de representacion de mensajer que permite: Cuerpos de Mensajes textuales que no sean en US-ASCII, diferentes formatos para mensajes no textuales, informacion en los "heaader" textuales en que no sean US-ASCII.[3]



Metodos de HTTP:

GET: recupera cualquire información que es identificada por ña èticion realizada (Request-URI). Puede ser cambiado a un Get Condicional si la peticion contiene un "IF-Modified-Since" en el header.[1]

HEAD: Este metodo es identico a GET, solo que este no regresa ninguna informacion relacionada con el cuerpo de la entidad.[1]

POST: Este metodo se encarga de enviar informacion al servidor.[1]


Errores de HTTP:

Dentro del protocolo HTTP existen dos tipos de errores, el primero son los errores de cliente, los cuales son identificables porque poseen un codigo que empieza con 4, cuando el cliente detecta este tipo de codigo deja de mandar cualquier tipo de data al servidor.Estos errores son: 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found.[1]

El otro tipo de error es el error de Servidor, caracterizado por empezar con un 5.Estos Errores son: 500 Internal server Error, 501 Not Implemented, 502 Bad GetWay, 503 Service Unavailable.[1]



XML:

XML es el acronimo para "Extensible Markup Languaje", que es un formato de texto flexible. Fue creado en un principio para cumplir los objetivos de la publicacion electronica a gran escala. XML es un lenguaje que se utiliza para describir otros lenguajes.[4]



XHTML

XHTML es una version de HTML que esta hecha de forma uqe XML la considera valida, esta version del lenguaje se utiliza por ser mas tecnica, lo que facilita el cambio de algunaas cosas, asi como el encontrar errores. Fue creado con el objetivo de crear una web semantica, osea que la informacion y la forma de presentar esta sean diferentes.[5]



HTML5

HTML5 es el estandar actual para la creacion de paginas web. HTML es un lenguaje de programacion que funciona con base a "Tags" los cuales estan encerrados por los simbolos de "<" y ">", los tags especificando cual es la funcion del texto u alguna caracteristica de este. Asi mismo los tags pueden tener otros atributos para reconocerlos, como una clase o un id. La principal diferencia entre la ultima version (HTML5) y la anterior es el hecho de que la mayoria de las etiquetas para dar formato a la pagina fueron cambiadas a otro lenguaje llamado CSS, sin embargo esto puede aumentar el nivel de informacion y trabajo de la aplicacion para comunicar esta informacion.



Java Script

Java Script es un lenguaje de programacion derivado de java que se utiliza para poder agregar un nivel de interactividad en las paginas web con el usuario. Java Script puede existir dentro del mismo documento de HTML (no es recomendable) o puede tener su propio documento, permitiendote enlazarlo a los documentos HTML que lo vayan a utilizar.



PHP

Las siglas de PHP representan a "HyperText Preprocessor, y eto es un lenguaje de codigo abierto que se puede integrar a HTML. La ventaja de este lenguaje es que el codigo se ejecuta dentro del mismo servidor, y unicamente se manda el resultado al cliente.[6]



Java Server Pages

El JSP es una tecnologia hecha para diseñar rapidamente paginas web dinamicas, esta herramienta permite cambiar el "layout" de la pagina sin generar cambios en la parte dinamica del contenido.[7]



AJAX

AJAX es unacronimo que significa "Asyncronous JavaScript And XML" la funcion de AJAX es ejecutar aplicaciones en el cliente mientras se mantiene una comunicacion con el servidor, permitiendo que la pagina se actualice sin la necesidad de que se vuelva a cargar.[8]




Preguntas



¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es un conjunto de redes interconectadas de manera inalámbrica o alámbrica a través de protocolos como lo son el TCP/IP, mientras que la World Wide Web es el sistema de distribución de documentos que se encarga de mandar paquetes a través del internet, explicándolo con una analogía el internet serian las carreteras que permiten a un usuario viajar entre ciudades (La World Wide Web).



¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Recupera datos de un recurso especifico.

HEAD: Pide una respuesta como la de GET pero sin el cuerpo de este ultimo.

POST: Envia una entidad a un recurso en especifico, causando un cambio en el servidor.

PUT: reemplaza datos del recurso actual con los del recurso pedido.

PATCH: Aplica modificaciones parciales a un recurso.

DELETE: Borra un recurso especifico.
